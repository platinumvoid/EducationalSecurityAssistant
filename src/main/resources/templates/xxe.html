XML External Entity (XXE)

SECURE METHOD

You can find the xml payload in the main.js file. If you are not using Linux change the file URL to any file you want on your system.

VULNERABLE METHOD

Protection:

The safest way to prevent XXE is always to disable DTDs (External Entities) completely. As you can see the secure method completely blocks the malicious payload. We have added a set of features you can use in your parser that can protect from such attacks. See XXE class for more information.

Vulnerability:

XXE is a type of attack against an application that parses XML input. This attack occurs when untrusted XML input containing a reference to an External Entity is processed by a weakly configured XML parser. Malicious payload: <!DOCTYPE foo [<!ENTITY xxe SYSTEM "file:///etc/passwd">]>
In this case scenario we can read any file on the server by just changing the file URL in the payload.
